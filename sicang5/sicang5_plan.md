# 四码仓颉（Sicang5）优化与重构计划 (v3)

您提出的这四点修改意见非常精确，完美解决了前一版中遗留的细节问题。本计划将指导接下来的代码优化。

## 1. 计划文档同步与更名
- 以后所有的设计改动都将在此文档（已从 `jiancang_plan.md` 更名为 `sicang5_plan.md`）中同步更新。这能确保我们的设计思路始终清晰有痕迹。

## 2. 剔除与全码完全相同的简码
- **问题**：像“这”（全码为 `yyk`），在提取三简时如果还是 `yyk`，就完全失去了简码的意义，反而会在生成的 `dict.yaml` 中产生一条无用的冗余数据。
- **规则更新**：在提取二简和三简时，增加严格校验：**如果提取出的简码与该字的 Sicang5 最终全码完全相同，则抛弃该简码**。简码必须**短于**全码才有资格进入简码表。

## 3. 二简词库的质量控制（生僻字过滤）
- **问题**：之前的二简有 623 个，其中充斥着像 `鍒(cd)`, `鑀(ce)` 这样字频极低甚至为 0 的冷僻字。这纯粹是为了占位而占位，没有实际意义。
- **规则更新**：为了保证二简的质量，我们将为二简池加入**字频前 3000** 的硬性门槛。只有排名前 3000 的常用字才有资格去竞选二简位。这样生成的二简表将绝对干净，全都是日常高频词。

## 4. 一简：原字根保护规则
- **问题**：原仓颉字根本身很多就是高频字（如一、人、中、日），如果盲目被其他字（如“的”、“是”）替换掉，会破坏形码用户的天然直觉。
- **规则更新**：引入**“原字根动态保护机制”**。
  - 对于每一个键，我们先查看该键原始字根（如 `m` 对应“一”）在语料库中的频率。
  - 只有当候选的高频字（且该高频字的编码中包含该字母）的频率 **大于原字根频率的 3 倍** 时，才能发生“夺舍”。
  - **效果演示**：
    - `m` (一)：频率极高（178万）。即使有其他字包含 `m`，也很难达到它的 3 倍（534万），所以 `m` 会稳定保留为“一”。
    - `h` (竹)：频率较低（6295）。“的”字的频率高达 240 万，轻松超越其 3 倍门槛，因此“的”会抢走 `h` 作为一简。
  - **优势**：完美兼顾了肌肉记忆（保住了常用字根）和极致提速（让低频字根退位让贤）。
