# 郑码规格草案

本文记录当前从仓库资料中提取出的郑码单字码表生成规则。规则以复现
`scripts/zhengma/data/chars.txt` 为目标；目前主流程已经明确，但旧郑码资料中仍有少量特殊取位规则尚未完全还原。

## 1. 数据来源

郑码单字码表由以下资料共同生成：

- `data/roots.txt`：字根表。每行格式为 `字根 组件码 [字根补码]`。
- `data/split.txt`：拆分表。每行格式为 `U+码位 字 拆分1 [拆分2 ...]`。
- `prototypes/one_code.txt`：一简单字原型。
- `prototypes/two_code.txt`：二简单字原型。
- `prototypes/three_code.txt`：三简单字原型。

`scripts/ids/ids.txt` 是通用 IDS 资料。`data/split.txt` 与它同源，但已经按郑码取形需要做了规范化，生成 `chars.txt` 时应以 `data/split.txt` 为准。

## 2. 输出格式

`data/chars.txt` 使用通用码表格式：

```text
字<TAB>编码
字<TAB>编码<TAB>造词码
```

没有造词码的条目只输出前两列，例如：

```text
⺀	tdaa
```

有造词码的条目输出三列，例如：

```text
的	nkrs	dv
的	d	dv
```

## 3. 字根表字段

`roots.txt` 的第二列是组件参与拆分取码时使用的组件码。第三列是字根字自身取码时追加的补码。

示例：

```text
土 b a
士 b aa
乙 yda
用 ld *
```

含义：

- `土` 作为组件时使用 `b`，作为字根字时输出 `ba`。
- `士` 作为组件时使用 `b`，作为字根字时输出 `baa`。
- `乙` 没有第三列，作为字根字时输出 `yda`。
- `用` 的第三列 `*` 是旧资料中的特殊标记；当前脚本尚未完全还原这类标记对末位补码的影响。

## 4. 拆分解析

拆分字符串按字根切分。花括号包裹的内容视为一个字根，其余按单个 Unicode 字符切分。

示例：

```text
十{具上}八 -> 十 / {具上} / 八
{巳}{七}攵山 -> {巳} / {七} / 攵 / 山
```

同一字在 `split.txt` 中可以有多个拆分备选。基础全码会按拆分备选顺序逐条生成，并对完整输出行去重。

## 5. 基础全码规则

设字拆为若干字根，每个字根在 `roots.txt` 中的组件码记为 `A`、`B`、`C`、`D`、`Z`；`Z` 表示末根。组件码可以是一位或多位。

### 5.1 字根字

规则：输出该字根的组件码加字根补码。

示例：

```text
一 [一 a] -> a
土 [土 b a] -> ba
士 [士 b aa] -> baa
乙 [乙 yda] -> yda
```

### 5.2 两根字

规则：拼接第 1 根组件码和第 2 根组件码，截取前 4 位；若结果只有 2 位，则补 `vv`。

示例：

```text
㐇 [九 qy / 乙 yda] -> qyyd
㐈 [入 oda / 乙 yda] -> oday
仲 [亻 n / 口 j / 丨 i] 不是两根字，见三根字和未复现规则。
```

### 5.3 三根字

规则：取第 1 根组件码、第二根首码、第三根组件码，拼接后截取前 4 位；若结果只有 2 位，则补 `vv`。

示例：

```text
真 [十 ed / {具上} lc / 八 o] -> edlo
的 [白 nk / 勹 ry / 丶 s] -> nkrs
㐁 [一 a / 囗 jd / 人 od] -> ajod
```

### 5.4 四根字

规则分两种情况：

- 若第 1 根组件码长度大于等于 2，取第 1 根组件码、第 3 根首码、第 4 根首码。
- 否则取第 1、2、3、4 根首码。

生成结果截取前 4 位；若结果只有 2 位，则补 `vv`。

示例：

```text
㐚 [工 bi / 𠃍 x / 口 j / 乙 yda] -> bijy
```

### 5.5 多根字

规则分两种情况：

- 若第 1 根组件码长度大于等于 2，取第 1 根组件码、倒数第 2 根首码、末根首码。
- 否则取第 1 根首码、第 2 根首码、倒数第 2 根首码、末根首码。

生成结果截取前 4 位；若结果只有 2 位，则补 `vv`。

示例：

```text
㮝 [木 f / 八 o / 人 od / 口 j / 卩 y] -> fojy
```

## 6. 造词码规则

造词码只写入 `〇` 和 CJK 及扩展区字。部首、笔画等较低码位字符没有造词码。

基础规则：

- 字根字：取该字根字自身编码的前两位；不足两位时补 `v`。
- 非字根字：取第一种拆分的第 1 根首码和第 2 根首码。

示例：

```text
〇 [〇 jaa] -> ja
㐁 [一 a / 囗 jd / 人 od] -> aj
真 [十 ed / {具上} lc / 八 o] -> el
```

若该字存在单字简码原型，则该字所有条目的造词码使用简码原型中的造词码覆盖。

示例：

```text
的	nkrs	dv
的	d	dv
着	ucl	ul
着	ul	ul
```

## 7. 简码合并

基础全码条目生成后，在同一字的基础条目之后合入单字简码原型：

1. `prototypes/one_code.txt`
2. `prototypes/two_code.txt`
3. `prototypes/three_code.txt`

合入时保留原型中的编码和造词码，并对完整输出行去重。

示例：

```text
的	nkrs	dv
的	d	dv

真	edlo	el
真	el	el

着	ucl	ul
着	ul	ul
```

## 8. 当前脚本

生成脚本为：

```powershell
python scripts/zhengma/generate_chars.py --output _tmp/zhengma_chars.generated.txt
```

校验当前复现程度：

```powershell
python scripts/zhengma/generate_chars.py --check --limit 20
```
