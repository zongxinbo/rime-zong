# 郑码学习指南

本文是对 `_tmp/zhengma/learning/郑码实用手册.pdf` 的学习向压缩整理，并结合本仓库
`scripts/zhengma/data` 与 `SPEC.md` 中已经整理出的规则。原 PDF 正文在当前环境中只能抽出
字形编码，不能可靠还原中文文本；本指南以手册章节结构、可读数据文件和现有生成规则为依据，
目标是帮助快速上手和复习规则，而不是替代完整手册。

## 1. 先抓住郑码的学习主线

郑码是形码：输入时先把字拆成字根，再把字根映射到键盘编码。真正的学习重点不是
直接背最终码，而是先理解“区码、主根、副根、补码”这几层。

建议顺序：

1. 先记 26 个区码，也就是每个键位代表的基本形类。
2. 再记每区的第一主根和第二主根。
3. 然后记副根，重点记副根相对主根多出来的位码。
4. 学会字根字自身编码和组合字取码的区别。
5. 最后熟悉简码和词语编码。

对应到本仓库文件：

- `data/roots.txt`：字根和组件码。
- `data/split.txt`：汉字拆分。
- `data/chars.txt`：最终单字码表。
- `SPEC.md`：当前脚本复现码表时使用的规则说明。

## 2. 区码、主根和副根

手册字根表的核心列大致是：

```text
区码 -> 主根 -> 副根 -> 高频字
```

其中“区码”就是键位的大类码，例如 `A` 区是一横，`B` 区和二横、土有关，`F` 区和木、西有关。

主根和副根要分开看：

- 第一主根通常直接用区码作组件码，例如 `一/a`、`土/b`、`王/c`、`木/f`、`口/j`。
- 第二主根通常是区码加位码，很多第二主根的位码是 `d`，例如 `二/bd`、`三/cd`、`十/ed`、`卜/id`、`冂/ld`。
- 副根也有自己的位码，例如 `示/bk`、`走/bo`、`車/fk`、`雨/fv`、`止/ii`、`见/lr`。
- 高频字是帮助记区码的口诀字，不等于该字一定用单码输入。

所以不能把“这个字根在某区”直接等同于“它的完整组件码就是区码”。区码只是第一层，第二码往往由主根/副根位置决定。

## 3. 字根码怎么看

`roots.txt` 每行大致是：

```text
字根 组件码 [字根补码]
```

例如：

```text
土 b a
士 b aa
二 bd
示 bk
丁 ai
木 f a
口 j
```

含义是：

- 字根参与组合字拆分时，使用第二列组件码。
- 字根本身作为单字输入时，可能要追加第三列补码。
- 没有第三列时，字根字通常直接用组件码。

所以：

```text
土 -> ba
士 -> baa
二 -> bd
示 -> bkaa
丁 -> ai
```

注意这里有两种码：

- **组件码**：参与别的字拆分时用，例如 `土` 作部件用 `b`，`示` 作部件用 `bk`。
- **字根字码**：这个字根本身作为一个字输入时用，例如 `土` 是 `ba`，`示` 是 `bkaa`。

这正是学习郑码时最容易混淆的地方。

## 5. 主根、副根和补码的关系

学习时可以把一个字根的编码拆成四层：

```text
区码：它属于哪个键位，也就是组件码第一码
位码：它在本区内的位置，通常是组件码第二码
组件码：它参与组合字拆分时用什么码
补码：它本身成字时追加什么
```

举例：

```text
土 b a   -> 作组件用 b，本字用 ba
士 b aa  -> 作组件用 b，本字用 baa
二 bd    -> 作组件和本字都用 bd
示 bk aa -> 作组件用 bk，本字用 bkaa
```

第一主根常常组件码只有一区码；第二主根和副根通常带完整的区位码；字根字自身还可能有补码。这几层不能混为一谈。

读字根表时可以按这几步看：

1. 先看这一行属于哪个区，比如 `B` 区、`F` 区、`L` 区。
2. 找第一主根，它通常就是本区最核心的形，例如 `B` 区的 `土`、`F` 区的 `木`。
3. 找第二主根，它通常有一个位码，例如 `二/bd`、`十/ed`、`冂/ld`。
4. 再看副根，副根一般写成“区码 + 位码”，例如 `示/bk`、`走/bo`、`雨/fv`。
5. 如果这个字根本身也是一个字，再看有没有补码，例如 `示 bk aa` 本字是 `bkaa`。

手册图里的红色字母可理解为区码，蓝色字母可理解为位码。学习时先记红色的大区，再记蓝色的区内位置，会比逐个背完整编码轻松。

## 6. 拆字原则

郑码取码依赖拆分。仓库中的 `data/split.txt` 已经把字拆到郑码字根层级，例如：

```text
真 -> 十 {具上} 八
的 -> 白 勹 丶
```

练习拆字时可以按下面顺序判断：

1. 能作为整体字根的先取整体。
2. 不能整体取时，再按部件拆成多个字根。
3. 花括号包住的是资料中定义的特殊字根，例如 `{具上}`。
4. 一个字可能有多个拆法，码表会按资料中的拆分备选生成。

实际生成码表时，以 `data/split.txt` 为准，不直接用通用 IDS。

## 7. 单字全码规则

设一个字拆成若干字根，各字根的组件码依次为：

```text
A B C ... Y Z
```

其中：

- `A` 是首根。
- `B` 是次根。
- `Y` 是次末根。
- `Z` 是末根。
- `a b y z` 表示这些根的首码。

郑码单字全码最多四码。统一理解是：先保留首根信息，再补足字尾信息；字根多时重点看次末根和末根。

### 7.1 字根字

字根本身作为单字：组件码加字根补码。

```text
一 -> a
土 -> ba
士 -> baa
乙 -> yda
```

### 7.2 两根字

两根字没有“次末根”和“末根”的取舍问题，直接取：

```text
首根组件码 + 末根组件码
```

再截取前 4 位；如果结果只有 2 位，则补 `vv`。

```text
㐇 = 九 / 乙 -> qyyd
㐈 = 入 / 乙 -> oday
```

### 7.3 三根及以上

三根及以上可以合并理解为：

```text
首根信息 + 字尾信息
```

字尾信息优先看“次末根 + 末根”。区别只在于首根信息够不够详细。

如果首根组件码有两位或以上，首根本身已经带了区码和位码，就取：

```text
首根组件码 + 次末根首码 + 末根首码
```

如果首根组件码只有一位，首根信息太少，需要补一个次根首码，就取：

```text
首根首码 + 次根首码 + 次末根首码 + 末根首码
```

三根字有一个特殊但很好理解的重合：次根同时也是次末根。为了保留末根信息，当前生成规则等价写作：

```text
首根组件码 + 次根首码 + 末根组件码
```

再截取前 4 位。

```text
真 = 十 / {具上} / 八 -> edlo
的 = 白 / 勹 / 丶 -> nkrs
```

四根字和多根字都按同一套心法处理：

```text
首根已足够具体：首根组件码 + 次末根首码 + 末根首码
首根只有一区码：首根首码 + 次根首码 + 次末根首码 + 末根首码
```

最终结果都截取前 4 位；如果结果只有 2 位，则补 `vv`。

例如：

```text
㐚 = 工 / 𠃍 / 口 / 乙
工=bi，首根已足够具体 -> bi + j + y -> bijy

副 = 一 / 口 / 田 / 刂
一=a，首根只有一区码 -> a + j + k + k -> ajkk

㮝 = 木 / 八 / 人 / 口 / 卩
木=f，首根只有一区码 -> f + o + j + y -> fojy
```

这比死记“三根、四根、多根”更好用：郑码真正想保留的是“开头怎么起形、结尾怎么收形”。中间很多根会被省略。

### 7.4 极度规则版

把单字全码压成判定树：

```text
先判断：这个字拆成几根？

1 根：
  输出：本根组件码 + 本根补码
  例外：补码为空时不追加；补码为 * 的旧资料特殊项，当前脚本尚未完全还原其真实作用。

2 根：
  输出：首根组件码 + 末根组件码
  后处理：截前 4 位；若只有 2 位，补 vv。

3 根：
  输出：首根组件码 + 次根首码 + 末根组件码
  后处理：截前 4 位；若只有 2 位，补 vv。
  注意：这里末根取完整组件码，不只是末根首码。

4 根：
  若首根组件码长度 >= 2：
    输出：首根组件码 + 第 3 根首码 + 第 4 根首码
  否则：
    输出：第 1 根首码 + 第 2 根首码 + 第 3 根首码 + 第 4 根首码
  后处理：截前 4 位；若只有 2 位，补 vv。

5 根及以上：
  若首根组件码长度 >= 2：
    输出：首根组件码 + 次末根首码 + 末根首码
  否则：
    输出：首根首码 + 次根首码 + 次末根首码 + 末根首码
  后处理：截前 4 位；若只有 2 位，补 vv。
```

换成统一记法：

```text
首根多码：首根组件码 + 次末根首码 + 末根首码
首根一码：首根首码 + 次根首码 + 次末根首码 + 末根首码
```

但有两个边界要单独记：

- 两根字：没有中间根，直接“首根组件码 + 末根组件码”。
- 三根字：次根也是次末根，但末根仍取完整组件码；这是为了避免尾部信息太少。

### 7.5 已知例外和边界

这些是学习或维护码表时需要单独注意的地方：

| 部分 | 规则 | 例外/边界 |
| --- | --- | --- |
| 字根表 | 第二列是组件码，第三列是字根字补码 | 第三列可为空；也可能是 `*`，例如 `用 ld *` 一类 `ld` 框形字根，旧资料含特殊标记，当前脚本未完全还原 |
| 主副根 | 第一主根常常只用区码，副根常常用区位码 | 不能按“同一区都是同一码”理解；例如 `土/b`、`二/bd`、`示/bk` 同在 B 区但组件码不同 |
| 字根字 | 本字码 = 组件码 + 补码 | 组件码不等于本字码；例如 `示` 作组件是 `bk`，本字是 `bkaa` |
| 拆分 | 以 `data/split.txt` 为准 | 同一字可能有多个拆分备选；会逐条生成并去重 |
| 花括号根 | `{...}` 视为一个字根 | 不能拆开逐字取码；例如 `{具上}` 是一个字根 |
| 两根字 | 首根组件码 + 末根组件码 | 若结果只有 2 位，补 `vv`；例如大量 `abvv` 类编码 |
| 三根字 | 首根组件码 + 次根首码 + 末根组件码 | 末根取完整组件码，这是统一公式的边界例外 |
| 四根字 | 看首根组件码长度分流 | 首根多码时不取第二根；首根一码时取第二根首码 |
| 多根字 | 看首根组件码长度分流，只保留首、次、尾 | 中间多数根会被省略，只保留次末根和末根 |
| 后处理 | 全码最多四码 | 所有基础全码都截前 4 位；只有 2 位时补 `vv` |
| 造词码 | 字根字取本字码前两位；非字根字取前两根首码 | 低码位部首、笔画等不写造词码；有简码原型时被原型造词码覆盖 |
| 简码 | 全码后合入一简、二简、三简 | 简码不是按全码规则现场推导，而是来自 `prototypes/*.txt` |
| 拆分资料 | 大陆宋体字形为主 | `㳤厯` 等取根分歧字、`㥁値塡巓悳愼槇眞鎭顚鷆` 等根序分歧字有兼容拆分；`乄` 对 `㇢` 有兼容处理 |
| 西夏字 | 增设专用字根 | 这些专用字根单独成字时补 `aa` 或 `aaa` |
| 当前复现 | 脚本规则能覆盖主流程 | 仍有旧郑码特殊取位、末根补码、异体拆分差异未完全复现，例如 `仲`、`佣`、`㟩` |

如果要极度机械地学习，建议每个字都按下面顺序审一遍：

```text
1. 查拆分：这个字有几个字根？有没有多个拆分？
2. 查每根组件码：首根是一码还是多码？
3. 按根数套公式：1 / 2 / 3 / 4 / 5+。
4. 截四码；若只有两码则补 vv。
5. 若是字根字，额外检查补码。
6. 查是否有简码原型；有则全码后追加简码。
7. 查造词码是否被简码原型覆盖。
```

### 7.6 不规则来源：简码和字根归并

郑码里有两类不能靠拆字公式推出的内容：

1. **简码原型**：为了高频输入指定的短码。例如 `的` 全码是 `nkrs`，但一简是 `d`。这个 `d` 不是拆字规则推导出来的，只能当作高频简码表记忆。
2. **字根归并**：为了同源字根、简繁字根或形近字根统一记忆，实际区码可能不按表层起笔推导。例如 `讠` 表层是点折开头，按笔形直觉容易想到 `W` 一类，但它归并到 `言/s`，实际是 `讠 s aaa`。

一简尤其要按表背，26 个键每键一个高频字：

```text
a 一    b 地    c 现    d 的    e 世    f 要
g 在    h 成    i 上    j 中    k 是    l 用
m 我    n 他    o 个    p 所    q 月    r 多
s 说    t 度    u 为    v 没    w 这    x 对
y 了    z 发
```

这些一简不是“理论编码”，而是输入层面的高频指定。

需要死记并持续维护的字根归并/分流项已单独生成到
`data/root_distribution_exceptions.md`。它由脚本生成：

```powershell
python scripts/zhengma/find_root_distribution_exceptions.py
```

当前清单包含 `讠/訁`、`乌/烏`、`鸟/鳥`、`鱼/魚`、`马/馬`、`车/車`、`门/門`
等容易按表层形态误判的组。学习策略不是推导，而是标成“人为归并/人为指定”：看见就背实际码。

## 8. 造词码

`chars.txt` 有些条目带第三列，这是造词码：

```text
的    nkrs    dv
的    d       dv
```

当前仓库规则中：

- 字根字的造词码取该字自身编码前两位，不足两位补 `v`。
- 非字根字取第一种拆分的第 1 根首码和第 2 根首码。
- 如果该字有单字简码原型，则造词码会被简码原型覆盖。

造词码主要用于词语编码，不必在刚入门时优先背。

## 9. 简码

郑码使用一简、二简、三简来降低常用字输入成本。本仓库的简码原型在：

- `prototypes/one_code.txt`
- `prototypes/two_code.txt`
- `prototypes/three_code.txt`

学习时可以先记高频字的简码，不必一次背全。全码能打出来以后，再把常用简码作为提速层叠加。

示例：

```text
的    nkrs
的    d

真    edlo
真    el

着    ucl
着    ul
```

## 10. 词语编码

手册中单独讲词语取码。实践上可以先按这个理解：

- 单字先有全码和简码。
- 词语编码依赖每个字的造词码或简码取位。
- 学习初期先把单字拆分和全码打稳，再练词语输入。

本仓库目前重点整理的是单字码表生成；词语规则如果要精确复现，应另行对照原方案资料。

## 11. 学习路线

推荐分五轮：

1. **认区**：每天按 5 到 6 个键位记区码和代表形。
2. **分主副**：每区先记第一主根、第二主根，再记高频副根。
3. **分用途**：同一个字根分别看组件码和字根字码。
4. **拆字算码**：拿常用字对照 `data/split.txt`，按规则手算，再查 `data/chars.txt`。
5. **提速**：加入一简、二简、三简和常用词。

练习时优先选常见字：

```text
的 真 着 中 人 木 王 口 日 月 心 言 水 火 金
```

每个字按“拆分 -> 每根组件码 -> 全码 -> 简码”四步过一遍。遇到字根字时，再单独检查有没有补码。
