# 灵仓

灵仓是基于仓颉五代字根的 A/B 互斥前缀码实验方案。

当前设计约束：

- A 区为主体字根码，不使用 `a/e/i/o/u`。
- B 区为 `a/e/i/o/u` 五个终止码。
- B 区取末仓颉字根发音的韵首，不取整字读音。
- 不复用灵明拆分、宏、专用 Lua 过滤或灵明词库。
- 反查使用仓库内 `pinyin_simp`。

初始韵首分组：

| B 码 | 末字根 |
| :--- | :--- |
| `a` | 大、山、難 |
| `e` | 戈、人 |
| `i` | 日、金、十、一、心、尸、廿、田 |
| `o` | 中、弓、手、口 |
| `u` | 月、木、水、火、土、竹、女、卜 |

当前 A 区初始合并：

| A 码 | 原仓颉字根 |
| :--- | :--- |
| `c` | 金、人 |
| `n` | 日、弓 |
| `w` | 山、田 |
| `x` | 戈、難 |
| `z` | 水 |

其余非 B 区字根尽量保留原键。设计上需要把 25 个仓颉字根映射到 21 个非 B 区字母，因此至少合并 4 对字根。

生成一简/二简原型：

```powershell
python scripts\lingcang\gen_shortcuts.py
```

默认按 `A 区 21 键 × B 区 5 键 - 25 个字根位` 自动生成 80 个二简码位。

输出：

```text
scripts/lingcang/prototypes/one_code.txt
scripts/lingcang/prototypes/two_code.txt
```
