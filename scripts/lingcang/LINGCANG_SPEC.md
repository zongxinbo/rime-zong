# 灵仓技术规范

本文档记录灵仓的设计目标、编码规则、当前 A/B 区映射与后续评估方法。

## 1. 设计目标

灵仓是基于仓颉五代字根的 A/B 互斥前缀码实验方案。

核心目标：

- 保留仓颉五代字根作为基础拆字来源。
- 使用 A/B 两个互斥键区，使编码自带字界。
- B 区只使用 `a/e/i/o/u` 五个字母，作为 1-3 码字的终止码。
- B 区取末仓颉字根发音的韵首，不取整字读音。
- 暂不设置额外简码层；先优化基础字根映射。

## 2. 键区划分

```text
B 区：a e i o u
A 区：b c d f g h j k l m n p q r s t v w x y z
```

A 区和 B 区互斥：

- A 区键只用于主体码。
- B 区键只用于短码终止。
- 不满 4 码的字以 B 区键结束。
- 4 码字全部由 A 区键组成，达到最大码长时结束。

## 3. 仓颉源码投影

灵仓从 `schemas/cangjie/cangjie5/cangjie5.dict.yaml` 读取仓颉五代码。

过滤规则与 Sicang5/Wucang5 对齐：

- 只处理单汉字。
- 跳过源码表中 `x` / `z` 开头的编码。
- 跳过无法映射到标准仓颉根的内部特殊编码。

原仓颉码先投影为四码：

```text
len(code) <= 4: code
len(code) > 4 : code[:3] + code[-1]
```

## 4. 编码规则

设原仓颉投影码为 `C1 C2 C3 C4`。

```text
原码 1 码：A(C1) + B(C1)
原码 2 码：A(C1) A(C2) + B(C2)
原码 3 码：A(C1) A(C2) A(C3) + B(C3)
原码 4 码：A(C1) A(C2) A(C3) A(C4)
```

其中：

- `A(C)` 是原仓颉字根到 A 区主体键的映射。
- `B(C)` 是原仓颉字根名读音韵首到 B 区终止键的映射。

例：

```text
日 a      -> A(日) + B(日) = zi
月 b      -> A(月) + B(月) = bu
明 ab     -> A(日) A(月) + B(月) = zbu
道 ythu   -> A(卜) A(廿) A(竹) A(山) = ythw
确 mrnbg  -> A(一) A(口) A(弓) A(土) = mrng
```

## 5. B 区韵首映射

B 区取“末仓颉字根名”的韵母首元音。

| B 码 | 原仓颉字根 | 读音依据 |
| :--- | :--- | :--- |
| `a` | 大、山、難 | da, shan, nan |
| `e` | 戈、人 | ge, ren |
| `i` | 日、金、十、一、心、尸、廿、田 | ri, jin, shi, yi, xin, shi, nian, tian |
| `o` | 中、弓、手、口 | zhong, gong, shou, kou |
| `u` | 月、木、水、火、土、竹、女、卜 | yue, mu, shui, huo, tu, zhu, nv, bu |

## 8. 评估命令

灵仓的 B 区尾码是显式上屏终止键，因此测评必须传入：

```powershell
python scripts\assess\summary.py --dict schemas\lingcang\lingcang.dict.yaml --commit-suffixes aeiou
```

A 区映射快速评估：

```powershell
python scripts\lingcang\evaluate_mappings.py
```

字根使用强度统计：

```powershell
python scripts\lingcang\analyze_roots.py
```

原始四码仓颉基准：

```powershell
python scripts\lingcang\gen_baseline_sicang.py
python scripts\assess\summary.py --dict _tmp\assess\sicang5_raw_cangjie5.dict.yaml
```

该基准直接从 `cangjie5.dict.yaml` 按四码规则截取，过滤 `x` / `z` 前缀编码，不包含简码、后缀 `z`、全码让位或其他排序优化。

## 9. 后续优化原则

A 区合并不应只靠手动试错。后续优先根据字根使用强度决定候选合并：

- 统计每个仓颉根在四码投影中的出现次数。
- 按综合字频加权统计每个根在各码位上的贡献。
- 优先避免合并高频、高区分力字根。
- 优先考虑低频、边缘根之间的合并。
- 对候选合并方案再用 `evaluate_mappings.py` 和 `summary.py` 验证。

## 10. 字根使用强度

- 源表：`schemas/cangjie/cangjie5/cangjie5.dict.yaml`
