# 四码仓颉（Sicang5）一简方案调优交接文档

如果您需要切换到其他 AI 工具（如 Gemini CLI）继续此任务，请提供此文档。

## 1. 当前任务背景
优化四码仓颉（Sicang5）的一简（One-Code）分配方案，目标是利用现代语料库实现“高频优先”与“编码直觉”的平衡。

## 2. 核心数据源
- **Dialogue 口语语料 (`dialogue_char_freq.txt`)**：新增口语字频，作为当前最核心的基底。
- **Subtlex 影视字幕语料 (`subtlex_char_freq.txt`)**：新增口语字频，提供大量生活对话参考。
- **知乎语料 (`zhihu_char_freq.txt`)**：现代网络常用语料。
- **北语大语料 (`blcu_char_freq.txt`)**：超大规模通用语料。
- **Essay 语料 (`essay-zh-hans.txt`)**：传统 Rime 语料。

## 3. 自动化分析工具
脚本位置：`scripts/cangjie/compare_one_codes.py`
该脚本实现了“最终版（Final Version）”算法，逻辑如下：
1. **口语优先**：默认采用 Dialogue 口语算法的结果作为基底。
2. **多源加权**：按 `Dialogue(6):Subtlex(5):Zhihu(4):BLCU(2):Essay(1)` 计算加权得分，确保口语字频权重第一。
3. **综合修正**：如果其它语料推荐的候选字加权得分显著高于基底字，则触发修正。
4. **首码保护机制**：
   - 优先保留符合首码直觉的分配（如 `a: 是`）。
   - 如果候选修正字不符合首码匹配而基底字符合，则需 **2.5 倍** 的频次得分差距才允许替换；否则只需 **1.2 倍**。
5. **换位逻辑 (Swap)**：修正时若发生冲突，自动交换按键位置，确保不产生重复或空缺。

## 4. 已确认的关键分配 (定稿建议)

- `a: 是` (首码保护生效，防止被“的”抢占)
- `w: 要` (成功通过频次压倒修正)
- `p: 也/他` (目前分配为“也”，用户反馈均可)
- `x: 其` (由换位逻辑产生，属于合理高频字)
- `t: 着` (成功通过共识修正)
