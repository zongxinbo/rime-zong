# 仓颉方案技术规范

本文档记录 Sicang5/Wucang5 的生成逻辑、简码分层策略与当前默认参数。

## 1. 任务背景

仓颉系列方案的目标是在保留仓颉五代取码直觉的前提下，用现代语料做高频优先排序与简码分配。Wucang5 是五码单字流方案；Sicang5 是四码单字流方案。

## 2. 核心数据源

- **Dialogue 口语语料**：`schemas/common/frequency/char/sc/dialogue_char_freq.txt`，权重 6
- **Subtlex 字幕语料**：`schemas/common/frequency/char/sc/subtlex_char_freq.txt`，权重 5
- **知乎语料**：`schemas/common/frequency/char/sc/zhihu_char_freq.txt`，权重 4
- **北语大语料**：`schemas/common/frequency/char/sc/blcu_char_freq.txt`，权重 2
- **Essay 语料**：`schemas/common/essay-zh-hans.txt`，权重 1

所有构建脚本通过 `cangjie_builder.get_weighted_frequencies()` 共享同一套加权字频，保证简码选择与最终字典排序使用同一份综合分。

## 3. 核心脚本

| 脚本 | 职能 |
| :--- | :--- |
| `core/gen_shortcut_1.py` | 一简校准生成与报告输出 |
| `core/gen_shortcut_2.py` | 二简净收益生成，支持固定数量、覆盖率、GB2312 保护 |
| `core/gen_shortcut_3.py` | 三简净收益生成，支持固定数量、覆盖率、GB2312 保护 |
| `core/gen_shortcut_4.py` | 四简生成，支持 `safe` / `balanced` / `aggressive` |
| `core/cangjie_builder.py` | 最终字典构建、排序与后缀消重 |
| `gen_wucang5.py` | Wucang5 一键构建入口 |
| `gen_sicang5.py` | Sicang5 一键构建入口 |

## 4. 简码原型文件

- 一简：`scripts/cangjie/prototypes/one_code.txt`
- 二简：`scripts/cangjie/prototypes/two_code.txt`
- 三简：`scripts/cangjie/prototypes/three_code.txt`
- 四简：`scripts/cangjie/prototypes/four_code.txt`（启用 `--s4` 时生成）
- z 补丁：`scripts/cangjie/prototypes/z_code.txt`

## 5. Wucang5 默认构建

默认命令：

```powershell
python scripts/cangjie/gen_wucang5.py
```

当前默认参数：

- `--s2-count 300`
- `--s3-count 800`
- `--s2-coverage 0`
- `--s3-coverage 0`
- `--protect-native-min-score 100000`
- `--no-s4`（默认关闭；可用 `--s4` 开启）
- `--s4-mode balanced`
- `--s4-count 1000`
- `--s4-level2-min-score 1000`

含义：

- 二简、三简默认按固定数量生成；只有当 `--s2-count 0` 或 `--s3-count 0` 时，对应覆盖率参数才参与控量。
- 二简默认取全码前两码，三简默认取全码前三码。
- 二简、三简的候选池按 Rime 默认 `extended_charset=常用` 状态过滤；Ext-A、Ext-B 及以上和兼容汉字等增广字符不会参与简码分配。
- `--protect-native-min-score` 同时控制原生二三码位保护门槛，以及二三简长码候选的最低入选综合字频。
- 四简默认关闭；需要实验四简时显式传入 `--s4`。
- 四简启用时只处理 GB2312 一级字和达到频率门槛的 GB2312 二级字，不把全 CJK 字集压入四码层。
- 四简启用时默认上限 1000 个；`--s4-count 0` 表示不做数量截断，由四简模式自身过滤规则决定数量。
- GB2312 二级字启用四简时默认需要综合字频达到 1000；`--s4-level2-min-score 0` 表示不过滤二级字。

常用命令：

```powershell
# 开启四简
python scripts/cangjie/gen_wucang5.py --s4

# 四简安全版
python scripts/cangjie/gen_wucang5.py --s4 --s4-mode safe

# 四简平衡版，启用四简时默认
python scripts/cangjie/gen_wucang5.py --s4 --s4-mode balanced

# 四简激进版
python scripts/cangjie/gen_wucang5.py --s4 --s4-mode aggressive

# 固定四简数量
python scripts/cangjie/gen_wucang5.py --s4 --s4-count 1000

# 调整 GB2312 二级字进入四简的门槛
python scripts/cangjie/gen_wucang5.py --s4 --s4-level2-min-score 1000

# 覆盖率控量
python scripts/cangjie/gen_wucang5.py --s2-count 0 --s2-coverage 0.85 --s3-count 0 --s3-coverage 0.90
```

## 6. 简码分层策略

### 6.1 一简与 z 补丁

一简由 `core/gen_shortcut_1.py` 单独生成校准表，生产构建脚本只消费 `one_code.txt`，不自动重算一简。

```text
判断依据 = 频率倍数 + 省码收益 + 字根位代价
```

当前校准在原高频一简表上做小范围调整：`g: 去 -> 地`、`l: 个 -> 中`、`p: 也 -> 心`、`r: 只 -> 和`。若字根字未被一简校准表选中，最终字典会过滤该字的原生一简入口，让它通过 `az/azz/bz...` 这类 z 字根码输入；若字根字在校准表中，则允许保留为一简。

### 6.2 二简

二简候选来自全码长度大于 2 的字：

- 默认取码：`full_code[:2]`
- 可选取码：首尾码
- 排序依据：净收益 `综合字频 × (原码长 - 2) - 原生码位代价`
- 默认模式：保护高频 GB2312 原生二码位，长码字可与其他原生二码位竞争
- 综合字频门槛：`--protect-native-min-score`，默认 `100000`
- 原生二码字是 GB2312 且达到门槛时受保护；长码字达到同一门槛才可入选二简
- `--no-protect-native`：允许长码字与原生二码字竞争，原主频次按 `1.5x` 计入净收益代价
- `--gb-only` 模式：仅 GB2312 字有资格，且 GB2312 原生二码位绝对保护

### 6.3 三简

三简候选来自全码长度大于 3、且未获得更短简码的字：

- 默认取码：`full_code[:3]`
- 可选取码：前两码 + 末码
- 排序依据：净收益 `综合字频 × (原码长 - 3) - 原生码位代价`
- 默认模式：保护高频 GB2312 原生三码位，长码字可与其他原生三码位竞争
- 综合字频门槛：`--protect-native-min-score`，默认 `100000`
- 原生三码字是 GB2312 且达到门槛时受保护；长码字达到同一门槛才可入选三简
- `--no-protect-native`：允许长码字与原生三码字竞争，原主频次按 `1.2x` 计入净收益代价
- `--gb-only` 模式：仅 GB2312 字有资格，且 GB2312 原生三码位绝对保护

### 6.4 四简

四简候选来自 GB2312、五码、且未获得 z/一简/二简/三简的字：

- 取码：`full_code[:4]`
- 启用后默认模式：`balanced`
- 默认关闭；开启方式：`--s4`
- 字集门槛：GB2312 一级字直接参与；GB2312 二级字需达到 `--s4-level2-min-score`

四简模式：

- `safe`：不压原生四码位，只接受没有原生四码冲突的候选。
- `balanced`：默认模式。若冲突原主已经有更短简码则放行；否则候选综合频次需超过活跃原主 `3.0x`。
- `aggressive`：GB 五码字全量截四码，保留内部重复，覆盖最大但候选压力也最大。

在 `safe` 与 `balanced` 下，四简会做码位去重与同字去重：每个四码位只保留一个最高价值字，同一个字也只保留一个四简。

## 7. 字典排序

最终字典使用纯 `字\t码` 格式，不写 weight。候选优先级依赖文件内排序：

1. 简码条目：`z` / 一简 / 二简 / 三简 / 四简（启用时）
2. 全码条目：同码内先按综合字频降序；如果首位已有简码，则让位给后续第一个达到常用门槛且没有简码的字
3. 后缀消重条目：自动追加的 `z` 后缀码

这个排序保证简码在自身码位上拥有绝对首选；全码位参考自然码的简码让位，只让出首选位，不把已有简码字沉底。默认常用门槛为综合字频 `1000`，可通过 `--fullcode-yield-min-score` 调整。

## 8. 后缀消重

`cangjie_builder.py` 会在构建末尾为重码候选字（跳过首选和已拥有简码的字）自动生成消重后缀直达码。支持以下两种模式：

### 8.1 默认模式（Z-后缀）
- 第二候选生成 `原码 + z`。
- 总码长不超过当前方案最大码长时才生成。
- 若第二候选字已经拥有首选简码入口则跳过，避免重复直达路径。

### 8.2 拼音规则模式（Pinyin Rule-based Suffix）
通过 `--suffix-rule pinyin` 参数启用。由于在生成后缀时我们排除了已获简码的字，且 4 码全码字不生成后缀，因此需要后缀的重码编码其原始长度必定在 1 码至 3 码之间。在这种低码位重码中，仅使用拼音首字母进行直觉式辅助消重。

消重直达码生成规则：

1. **GBK 字集范围过滤**：仅为属于 **GBK 字符集**（`char.encode('gbk')` 成功）的汉字生成消重后缀，避开日常打字根本遇不到且不知读音的数十万个 Extension B/C/D/E 罕见生僻字，以保持词典轻量和纯净。
2. **免加后缀条件**：
   - 候选列表中第 1 候选（首选）不加后缀。
   - 如果候选字已经拥有简码（一简、二简、三简等），则不加后缀，避免冗余。
   - 若当前重码编码的长度已经达到或超过方案的最大码长（例如四码方案下已经是 4 码的字），则**暂不补任何后缀**。
3. **拼音辅助消重规则**：
   - 对于符合条件的第 2 候选及以后的字，读取 `scripts/shuangpin/prototypes/chars.txt` 里的拼音信息，提取其**拼音首字母**作为消重后缀（例如 `头` $\to$ `ykt`，`文` $\to$ `ykw`；`行` $\to$ `homnh`；`井` $\to$ `ttj`）。
   - **兜底**：若拼音表中无此字读音，则不生成后缀，用户打原码在候选框选择即可。

这层逻辑在不改变已有简码优先级的前提下，大大降低了全码模式下的选重压力。

### 8.3 结构规则模式（Structure Rule-based Suffix）

通过 `--suffix-rule structure` 参数启用。本模式使用汉字的**间架结构**（由 IDS 字根描述序列首字符决定）作为消重后缀，完全不依赖读音，适合不知读音也能盲打识形的仓颉用户。

#### 结构数据来源

结构信息读取自 `scripts/ids/ids.txt`，该文件每行格式为：

```
U+xxxx <字符> <IDS描述>
```

取每个字 IDS 描述的**首字符**（IDS 运算符）来判断结构类别。

#### 结构分类与键位映射

| 结构类别 | IDS 运算符 | 后缀键 | 手位 | 记忆关联 |
| :--- | :--- | :---: | :---: | :--- |
| 左右 / 左中右 | `⿰` `⿲` | `a` | 左 | **A** 对应键盘最左侧基础键 |
| 上下 / 上中下 | `⿱` `⿳` | `s` | 左 | **S**tack（堆叠） |
| 包围 / 半包围 | `⿴` `⿵` `⿶` `⿷` `⿸` `⿹` `⿺` `⿻` | `d` | 左 | **D**oor（门框、围合） |
| 独体 / 单一字根 | 其他（非 IDS 运算符开头） | `f` | 左 | **F**ree（自由独立） |

> **说明**：结构后缀键固定为 **`a s d f`（即 "asdf"）**，无需额外配置映射表。所有四个结构后缀键均位于**左手**，配合四码仓颉本身右手偏重（约 58%）的键位分布，显著改善整体手指负荷平衡。

#### 生成规则

1. **免加后缀条件**（同拼音模式）：
   - 候选列表中第 1 候选（首选）不加后缀。
   - 已拥有简码的字不加后缀。
   - 当前重码编码长度已达到或超过方案最大码长时不加后缀。
2. **无损安全约束**（结构模式专有）：
   - 若拟生成的后缀码（`原码 + 结构键`）在词典中**已被其他条目占用**，则放弃生成，不制造新的二次重码，直接让用户在候选框选字。
3. **兜底**：若 `ids.txt` 中无此字的结构信息，则不生成后缀。

> **注**：结构模式不做 GBK 字集过滤。纯形识别不依赖读音，凡是 `ids.txt` 中有记录的字均可生成结构后缀，生僻字同样适用。

#### 典型示例

| 字 | 原码 | 结构 | 后缀键 | 结构直达码 |
| :---: | :---: | :---: | :---: | :---: |
| 杠 | `dm` | 左右 | `a` | `dma` |
| 杢 | `dm` | 上下 | `s` | `dms` |
| 扌 | `dm` | 独体 | `f` | `dmf` |
| 井 | `tt` | 包围 | `d` | `ttd` |
| 卌 | `tt` | 独体 | `f` | `ttf` |
| 匕 | `um` | 独体 | `f` | `umf` |
| 屵 | `um` | 上下 | `s` | `ums` |
| 屸 | `um` | 左右 | `a` | `uma` |

经数据测算（针对 Sicang5，≤3 码全部冲突字，不限 GBK，共 5922 个），无损安全约束下可生成约 **3066 个** 独占无碰撞的结构直达码，覆盖约 **51.8%** 的目标冲突字，全局二次重码几乎不增加。

## 11. 词组取码动态配额

Sicang5 语句流采用动态配额算法，根据单字码长分配 4 码位。

| 词长 | 结构配额 | 取码明细 | 示例 |
| :--- | :--- | :--- | :--- |
| 二字词 | 2 + 2 | A首+A尾 + B首+B尾 | 实际 -> pmnf |
| 二字词 | 1 + 3 | A首 + B首+B次+B尾 | 中国 -> lwim |
| 二字词 | 2 + 1 | A首+A尾 + B首 | 某个 -> mwg |
| 三字词 | 2 + 1 + 1 | A首+A尾 + B首 + C尾 | 实际上 -> pmnm |
| 三字词 | 1 + 2 + 1 | A首 + B首+B尾 + C尾 | 输入法 -> johy |
| 三字词 | 1 + 1 + 2 | A首 + B首 + C首+C尾 | 这种人 -> yhoo |
| 四字词+ | 1 + 1 + 1 + 1 | A首 + B首 + C首 + Z首 | 社会保障 -> ioof |

原则：遵循仓颉五代顺位，动态消除同字内冗余码，首字不超过 2 码。
