# 仓颉方案技术规范 (CANGJIE_SPEC)

本文档记录了仓颉系列方案的设计逻辑、自动化调优算法以及核心架构原则。

## 1. 任务背景
优化仓颉系列方案（Sicang5/Wucang5）的简码分配方案，目标是利用现代语料库实现“高频优先”与“编码直觉”的平衡。

## 2. 核心数据源
- **Dialogue 口语语料 (`frequency/char/dialogue_char_freq.txt`)**：核心口语字频（源自北语大语料库加工），确保高频对话词汇优先。
- **Subtlex 影视字幕语料 (`frequency/char/subtlex_char_freq.txt`)**：基于影视字幕的日常频次，反映口语视听习惯，覆盖常用生活用语。
- **知乎语料 (`frequency/char/zhihu_char_freq.txt`)**：现代网络常用语料，提供更具时代感的频次分布。
- **北语大语料 (`frequency/char/blcu_char_freq.txt`)**：超大规模平衡语料库，提供基础字频基底。
- **Essay 语料 (`frequency/word/essay-zh-hans.txt`)**：传统 Rime 基础语料库。

## 4. 词组取码动态配额规则 (Dynamic Quota Logic)
Sicang5 采用动态配额算法，根据单字码长动态分配 4 码位。

| 词长 | 结构配额 | 取码明细 | 典型示例 |
| :--- | :--- | :--- | :--- |
| **二字词** | 2 + 2 | A首+A尾 + B首+B尾 | 实际 (pkhm + nlf) -> pmnf |
| | 1 + 3 | A首 + B首+B次+B尾 | 中国 (l + wirm) -> lwim |
| | 2 + 1 | A首+A尾 + B首 | 某个 (mow + g) -> mwg |
| **三字词** | 2 + 1 + 1 | A首+A尾 + B首 + C尾 | 实际上 (pkhm + nlf + ym) -> pmnm |
| | 1 + 2 + 1 | A首 + B首+B尾 + C尾 | 输入法 (jokon + oh + eiy) -> johy |
| | 1 + 1 + 2 | A首 + B首 + C首+C尾 | 这种人 (ypt + hdyj + o) -> yhoo |
| **四字词+** | 1+1+1+1 | A首 + B首 + C首 + Z首 | 社会保障 (if+omr+ord+fylj) -> ioof |

*原则：遵循仓颉5顺位，动态消除同字内冗余码，首字不超过2码（不反自身）。*

## 6. 自动化简码策略与覆盖率 (Auto-Coverage Strategy)

在五码仓颉（Wucang5）的单字方案中，为了平衡“极致打字手速”与“免视觉依赖的盲打心流”，我们引入了基于累计字频的动态覆盖率（Auto-Coverage）算法。默认最优参数组合为：**二简 0.90 + 三简 0.90**。

### 6.1 黄金参数：0.90 阈值与肌肉记忆
汉字存在极端的长尾效应。通过计算语料库累计分布，`0.90` 阈值划定了一条“及格线”（涵盖了贡献日常 90% 文本量的高频字池）。
*   **一简**：25 个固定最高频字。
*   **二简 (0.90)**：约 275 个字（淘汰打不过原主及同码碰撞的字）。
*   **三简 (0.90)**：约 324 个字。
*   **总计**：约 624 个人工简码字。

**理论支撑**：在中文形码输入法（如五笔）的 40 年演进史中，600~700 个简码字是人类“无意识肌肉记忆”的物理上限。这 624 个核心简码字通过高频刺激，能够完美转化为肌肉条件反射。超过此数量的简码通常会被大脑遗忘，导致“打字停顿看屏幕”的视觉依赖。

### 6.2 竞争淘汰与前缀碰撞
算法的核心并非强行凑数，而是**极度尊重键位原主**。
*   当多个高频字竞争同一个简码坑位（如前三码同为 `ABC`）时，只有绝对频次最高或达到原主频次 1.2/1.5 倍的字才能胜出。
*   **牺牲与兜底**：大量落榜的超高频字（如被“大”挤掉的“太”）会被强制下放为全码。这虽然牺牲了简码字面的理论覆盖率，但保证了拆字逻辑的连贯性。对于这部分字，用户直接打穿五码（静重极低）即可实现免看屏幕的盲打兜底。

### 6.3 最终的真实打字体验（87.61% 极速覆盖）
当用户日常打字时，只要敲击 1~3 下键盘且作为第一候选字上屏的，包含两类字：
1. **人工生成的 600 多个简码字**。
2. **仓颉原生短码高频字**（其全码本身就只有 1~3 码，且在竞争中被算法保护的原住民）。

经全量字库统计，这两部分字合兵一处，**真实覆盖了日常输入文本的 87.61%**。
这意味着近 9 成的输入可通过 1~3 键极速上屏，而剩下的 1 成多则由五码兜底。这套 `Wucang5 + 0.90双参数` 体系代表了单字形码输入法在速度与确定性上的巅峰平衡。
