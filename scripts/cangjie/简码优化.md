# Wucang5 低重码顺位简码优化方案

## 核心设计目标

在仓颉五代（最高 5 码）的基础上，强制要求所有 GB2312 汉字编码长度 ≤ 4 码。
采用"顺位前缀取码"与"GB2312 绝对保护机制"生成 2、3、4 简码，
全码采用"位置退避"策略，保障顺延打字的极致流畅与低重码。

## 设计原则

1. **顺位前缀**：所有简码必须是全码的绝对前缀 (`full_code[:N]`)
2. **GB2312 绝对保护**：如果某码位有原主字（全码长度恰好等于该码长），且属于 GB2312，绝对禁止抢占
3. **GB2312 资格门槛**：只有 GB2312 字才有资格获得简码
4. **繁体字透传**：非 GB2312 字保持仓颉五代原编码不变
5. **层级排他**：已在更短层获得分配的字，不参与更长层级竞争

## 四层简码规则

### 一简 (S1)：25 字
手动精选最高频字，存放于 `one_code.txt`。

### 二简 (S2)：~286 字
- 取码：`full_code[:2]`
- 仅占据"空槽"（该 2 码编码上没有任何 GB2312 原主字）
- 每个空槽选该前缀下最高频的 GB2312 长码字

### 三简 (S3)：~2,376 字
- 取码：`full_code[:3]`
- 仅占据"空槽"，排除已获 S2 的字
- 每个空槽选该前缀下最高频的 GB2312 长码字

### 强制四简 (S4)：~1,332 字
- 对象：全码 = 5 码且未在 S1/Z/S2/S3 获得分配的 GB2312 字
- 取码：`full_code[:4]`
- 无竞争保护，允许物理重码，依靠字频排序

## 位置降权（Positional Demotion）

字典不使用 weight 列，保持纯 `字\t码` 格式以保证跨平台兼容性。
"降权"通过文件内排列顺序实现：

同一编码下的候选排序：
```
① 简码条目（Z > S1 > S2 > S3 > S4）
② 原生全码条目（该字无更短简码）→ 按字频降序
③ 退避条目（该字已有更短简码）→ 按字频降序
```

## 字典头部格式

```yaml
name: wucang5
version: '2026-05-05'
sort: by_weight
```

不包含 `vocabulary`、`max_phrase_length`、`min_phrase_weight`，为纯单字流模式。

## 生成脚本

```bash
# 在 scripts/cangjie/ 目录下运行
python gen_wucang5.py
```

## 验证结果

- GB2312 最短编码超 4 码的字数：**0**
- 动态重码率：0.3051% → 0.5685%（+0.26%，代价极小）
- GB2312 码长分布：1码=51字，2码=608字，3码=3550字，4码=2554字
